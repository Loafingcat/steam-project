🟡 슬라이드 2 — 흥행 예측

두 번째 모델은 흥행 예측 모델입니다.

앞서 숨은 명작 탐지가 "좋은데 덜 알려진 게임"을 찾는 문제였다면, 이번에는 조금 더 직접적으로 "이 게임이 흥행할 가능성이 있는가?"를 예측하는 문제입니다.

입력 데이터는 동일하게 가격, 장르, 플랫폼, 지원 언어 등 게임 출시 시점에 알 수 있는 정보들입니다.

다만 목표가 달라졌습니다.

숨은 명작 탐지는 평가가 좋은 게임을 찾는 것이었다면, 흥행 예측은 실제로 많은 유저를 확보할 수 있는 게임을 찾는 것이 목적입니다.

이 모델 역시 성공작 비율이 18%에 불과한 데이터 불균형 문제가 있었습니다. 초기 랜덤 포레스트 모델은 실제 흥행작을 찾아내는 성능인 Recall이 0.28에 머물러, 

흥행작의 70% 이상을 놓치는 한계가 있었습니다. 이를 해결하기 위해 1대1 다운샘플링을 적용했고, 그 결과 Recall을 0.78까지 크게 끌어올렸습니다.

이후 다양한 알고리즘으로 비교 실험을 진행했는데, "숨은 명작 탐지" 모델과는 완전히 다른 결과가 나왔습니다. 숨은 명작 탐지에서는 트리 기반 머신러닝이 우세했던 반면, 

흥행 예측에서는 딥러닝 MLP 모델이 Recall 0.82, F1-Score 0.61로 가장 높은 성능을 보여주었습니다.

이러한 성능 차이는 데이터의 성격과 밀도 때문이었습니다.
초기 정형 피처 위주의 꽉 찬 밀집(Dense) 데이터 환경에서는 변수 간 관계를 송곳처럼 쪼개 나가는 LightGBM이 우수했습니다. 

하지만 고도화 과정에서 게임 설명글을 TF-IDF 텍스트 피처로 추가하자, 대부분이 '0'으로 채워지는 극단적인 희소(Sparse) 데이터 환경이 되었습니다.

이때 트리 모델은 분기 조건이 흔들리며 한계를 보인 반면, 심층 신경망인 MLP는 행렬 연산을 통해 수치 데이터와 텍스트의 맥락을 고차원 공간에서 유기적으로 융합해 내며 희소성을 극복했습니다.

또한 모델 개발 과정에서 중요한 문제도 발견했습니다.

초기 구현에서는 Test Set 성능을 기준으로 최적 모델을 선택하고 있었는데, 이는 테스트 데이터를 모델 선택에 간접 참조하는 '데이터 누수', 즉 데이터 리키지 위험이 있었습니다.

그래서 Validation Set을 별도로 분리하고, 입력 데이터를 StandardScaler로 정규화한 뒤, Validation Loss가 가장 낮은 모델만 저장하도록 학습 구조를 수정했습니다.

그 결과 Accuracy는 소폭 감소했지만, 실제 서비스 환경에 가까운 보다 신뢰성 있는 성능을 확보할 수 있었습니다.

이 실험을 통해 정형 데이터 환경이라도 텍스트 결합으로 인한 데이터의 희소성 같은 문제 특성에 따라, 머신러닝보다 딥러닝이 더 좋은 결과를 낼 수 있다는 점을 확인할 수 있었습니다.

3. 검색 모델

마지막 모델은 자연어 검색 기능으로 이전의 모델과는 성격이 다릅니다.

가격, 장르 같은 숫자와 범주형 데이터, 즉 정형 데이터를 다뤘다면 이번에는 게임 설명문과 태그 같은 텍스트 데이터를 활용했습니다.

이 모델을 만들게 된 계기는 앞서 말씀 드렸다시피 스팀 검색 기능의 한계 때문이었습니다.

"잔잔하게 힐링할 수 있는 게임 없을까?", "스토리가 좋은 게임 하고 싶은데?"처럼 자연어로 검색하는 기능을 하는 모델을 구현해봤습니다.

이 검색 시스템은 두 가지 방식으로 구현해서 비교했습니다.

첫 번째는 머신러닝 기반의 TF-IDF 방식입니다. 

쉽게 말하면 단어를 직접 매칭하는 방식입니다. 

사용자가 "horror"라고 검색하면 게임 설명이나 태그에 "horror"라는 단어가 포함된 게임을 찾아줍니다.

구현이 간단하고 속도가 매우 빠르며, 짧은 영어 키워드 검색에서는 상당히 좋은 성능을 보였습니다.

하지만 명확한 한계도 있었습니다.

예를 들어 사용자가 "힐링 게임"이라고 검색했을 때, 게임 설명에는 "relaxing"이라고 적혀 있다면 두 단어가 같은 의미임에도 불구하고 

서로 다른 단어로 인식하기 때문에 검색 결과에 나타나지 않습니다.

이 문제를 해결하기 위해 두 번째 방식인 딥러닝 기반 의미 검색을 적용했습니다.

이 방식은 문장을 숫자 벡터로 변환한 뒤 의미적 유사성을 비교합니다. 

따라서 "힐링 게임"과 "relaxing game"처럼 표현은 다르지만 의미가 비슷한 문장도 서로 가깝게 인식할 수 있습니다.

즉 한국어로 검색하더라도 영어로 작성된 게임 설명을 찾아낼 수 있고, 단순 키워드 검색보다 훨씬 자연스러운 검색이 가능해집니다.

실험 결과도 두 방식의 특징을 잘 보여주었습니다.

★: 와 진짜 편한데요. (박수) 그 편한걸 스팀은 안해주더라고요.

12. "horror", "puzzle" 같은 짧은 영어 키워드 검색에서는 TF-IDF가 약 0.64의 성능을 기록하며 좋은 결과를 보였습니다. 

특정 단어가 그대로 포함되어 있는지를 확인하는 문제에서는 단어 매칭 방식이 매우 효과적이기 때문입니다.

반면 한국어 검색이나 자연어 문장 검색에서는 TF-IDF가 사실상 제대로 동작하지 못했습니다. 같은 의미라도 사용된 단어가 다르면 연결할 수 없기 때문입니다.

반대로 딥러닝 기반 의미 검색은 이러한 상황에서 훨씬 안정적인 검색 결과를 보여주었습니다.

그래서 최종적으로는 두 방식을 결합한 하이브리드 검색 시스템을 구성했습니다.

검색어가 짧은 영어 키워드인 경우에는 빠른 TF-IDF 검색을 사용하고, 한국어나 긴 문장 형태의 검색어가 입력되면 의미를 이해할 수 있는 딥러닝 검색을 사용하도록 설계했습니다.

즉 사용자는 어떤 방식인지 신경 쓸 필요 없이 검색어만 입력하면 상황에 맞는 검색 엔진이 자동으로 선택됩니다.

또한 검색 결과를 단순히 텍스트 유사도만으로 정렬하지 않았습니다.

검색어와의 관련성뿐 아니라 리뷰 수, 평점과 같은 실제 사용자 반응도 함께 반영하여, 

단순히 비슷한 게임이 아니라 실제로 좋은 평가를 받은 게임이 상위에 노출되도록 설계했습니다.

★: 평가가 죠진 게임은 안보인다는게 더 마음에 드는군요!

13. - 전체 마무리

정리하면, 같은 스팀 데이터셋으로 세 가지 문제를 풀어보면서 머신러닝과 딥러닝이 항상 같은 결과를 내는 것은 아니라는 점을 확인할 수 있었습니다.

숨은 명작 탐지에서는 XGBoost 기반 머신러닝 모델이 가장 좋은 성능을 보였고, 흥행 예측에서는 딥러닝 MLP가 가장 높은 탐지 성능을 기록했습니다.

반면 텍스트 기반 자연어 검색에서는 의미를 이해할 수 있는 딥러닝 모델이 머신러닝보다 훨씬 우수한 결과를 보여주었습니다.

결국 이번 프로젝트를 통해 얻은 결론은 특정 모델이 항상 더 뛰어난 것이 아니라, 데이터와 문제의 특성에 따라 적합한 모델이 달라진다는 것입니다.

저희 팀은 동일한 데이터셋 안에서 정형 데이터 분류, 불균형 데이터 분류, 자연어 검색이라는 세 가지 문제를 직접 비교 실험하며 이를 확인할 수 있었습니다.


이상으로 모델에 대한 설명을 마무리하고 모델 시연을 보여드리겠습니다.
