### Machine Learning: Steam Game Success Prediction

이 프로젝트는 스팀 게임 데이터를 기반으로 특정 게임이 흥행(Hit)할지 여부를 예측하는 분류 모델을 구축하는 데 목적이 있습니다. 83개의 정형 피처를 활용하여 비즈니스 가치가 높은 흥행작 탐지 성능을 개선하는 과정에 집중했습니다.

---

#### 1. 문제 정의 및 데이터 불균형 해결 (Baseline: Random Forest)

초기 모델 구축 단계에서 랜덤 포레스트를 활용했으나, 데이터 불균형 문제로 인해 모델이 비흥행작 위주로 학습되는 편향이 발생했습니다.

* **문제 상황**: 전체 데이터 중 성공작(1)의 비율이 약 18%에 불과하여, 모델이 대다수인 실패작(0)으로 예측하는 경향이 매우 강했습니다.
* **해결 전략**: **데이터 레벨 다운샘플링(Downsampling)** 적용
    * 비흥행작(0) 데이터를 성공작(1) 데이터 수와 동일하게(1:1) 맞추는 무작위 비복원 추출을 수행했습니다.
    * **이유**: 단순 정확도(Accuracy)의 착시를 제거하고, 플랫폼 기획의 핵심인 '흥행작 탐지 성능(Recall)'을 정량적으로 확보하기 위함입니다.

---

#### 2. 모델 고도화: 알고리즘 교체 (LightGBM)

기본 모델에서 확보한 데이터 균형을 바탕으로, 더 정교한 패턴 학습을 위해 LightGBM 알고리즘으로 모델을 교체했습니다.

* **모델 선정 이유**: 
    * **패턴 학습 능력**: 랜덤 포레스트보다 복잡한 데이터 관계를 학습하는 데 효과적인 부스팅 알고리즘을 도입하여 정밀도와 재현율을 동시에 개선하고자 했습니다.
    * **효율성**: 데이터가 증가하더라도 연산 자원을 효율적으로 사용하여 빠른 모델 학습 및 배포가 가능합니다.
* **실험 결과**: 
    * 재현율(Recall) 0.80, 정밀도(Precision) 0.46을 확보하여 이전 모델 대비 전반적인 예측 균형(F1-score 0.58)을 높였습니다.

---

#### 3. 딥러닝 기반 검증 및 과적합 제어 (PyTorch MLP)

동일한 1:1 균형 정형 데이터셋 환경에서 트리 기반 모델 외에 **심층 신경망(MLP)**이 복잡한 비선형 경계선을 찾아낼 수 있는지 추가 검증을 진행했습니다.

* **실험 및 트러블슈팅 (과적합 제어)**:
    * 초기 MLP 모델 설계 시, 후반부 에폭으로 갈수록 Train Loss는 감소하지만 Test 정확도가 급락하거나 롤러코스터처럼 요동치는 오버슈팅(Overshooting) 현상이 발생했습니다.
    * 이는 피처가 제한된 정형 데이터 환경에서 모델이 특정 노이즈를 암기하여 발생한 전형적인 과적합 신호였습니다.
    * 이를 해결하기 위해 신경망 내부의 규제(Regularization)를 강화하고자 **드롭아웃(Dropout) 비율을 기존 0.3 / 0.2에서 0.4 / 0.3으로 상향 조정**했습니다.
* **인프라적 안전장치**:
    * 학습 루프 내에 **최적 검증 손실 기준 베스트 모델 저장 로직(`_best.pth`)**을 설계하여, 후반부 에폭의 변동성에 영향을 받지 않고 가장 일반화 성능이 뛰어난 시점의 가중치를 파일 시스템에 격리·확보했습니다.
* **실험 결과**:
    * 드롭아웃 수치 조절 후 후반부 폭락 현상이 완벽히 방어되었으며, **3에폭 시점에서 과적합 없는 클린한 최고 검증 정확도 85.93%**를 기록하여 모델의 순수 체급과 안정성을 증명했습니다.

#### 향후 계획
비즈니스 목적에 따른 임계값(Threshold) 조정: 현재 모델은 기본 확률 값 0.5를 기준으로 예측하고 있습니다. '흥행작'을 놓치지 않는 것이 비즈니스적으로 더 중요하다면, 임계값을 낮추어 재현율(Recall)을 극대화하는 최적화 실험을 추가할 예정입니다.

모델 해석력 보완: 어떤 게임 피처(Feature)가 흥행에 가장 큰 영향을 미쳤는지 모델의 변수 중요도(Feature Importance)를 시각화하여, 단순 예측을 넘어 흥행 요인을 분석하는 인사이트 도출 단계로 확장하고자 합니다.
